Introduction

Credit card fraud is a significant threat in financial systems, causing billions in losses each year. Traditional models often fail when malicious actors slightly manipulate features. In this project, we explore the use of adversarial examples to simulate realistic fraud attempts and evaluate model robustness.

Workflow

Our project follows a structured pipeline from raw data to adversarial evaluation.

Dataset and EDA

We use the XYZ Credit Card Transactions dataset. This dataset contains N rows and M features including transaction amount, time, merchant category, etc.

  • EDA includes class imbalance visualization and time distribution
  • Preprocessing: normalization, outlier filtering, and categorical encoding

Models and Results

We benchmark several models: Logistic Regression, Random Forest, and an Adversarially-trained Neural Network. Performance is compared using AUC and F1 Score.

Our adversarial model improved recall by 15% compared to baseline, without sacrificing precision.
